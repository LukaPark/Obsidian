- <a> 와 next/link 의 차이점.
	- <a> 태그는 페이지 라우팅에 전체 페이지 새로고침을 수반.
	- <Link> 태그는 <a> 태그와 유사하게 href 속성을 가지지만, 
	- 페이지 새로고침을 진행하지 않고 라우팅.

# 자동 코드 분할 및 pre-fetching
	- 탐색 경험을 향상시키기 위해. NextJs 는 경로 세그먼트 별로 앱을 자동으로 코드 분할을 진행. 기존 React SPA 와 는 다름. 브라우저는 초기 로드시 모든 앱 코드를 로드.
	- 경로 별로 코드를 분할 한다는 것은 페이지가 격리된다는 의미인데, 특정 페이지에서 오류가 발생하더라도 앱의 나머지 부분은 계속 작동.
	- <Link> 구성요소가 브라우저의 뷰포트에 나타나면, NextJs는 백그라운드에서 연결된 경로에 대한 코드를 자동으로 가져옴. *pre-fetching
	- 그렇기 때문에 사용자가 링크를 클릭 시 이미 백그라운드에 코드가 로드되어 있어 페이지 전환이 즉각적으로 이루어 짐. 


# Show Actived Link *활성 링크 표시
	- 사용자가 현재 존재하는 페이지를 알려주는 활성 링크를 사용하기 위해.
	- usePathname() hook.
	- 클라이언트 구성요소로 전환 필요, "use client"
	- usePathname() 과 link.href *이동 될 경로* 가 같은 경우.
